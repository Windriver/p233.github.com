Syntax Highlighting for Sass

用过几套 Sass 代码高亮插件，效果都不理想，下决心自己写了一个，很多地方参考了 nathos's sass-textmate-bundle 与默认的 CSS 高亮包。经过几轮修改已经非常完美了，欢迎大家使用，在 Package Control 中搜索 shs 就可以发现这款插件了，特色功能包括：

  1. 采用匹配语句结构的方式代替匹配关键词，因此不会出现某些属性名、属性值无法高亮的情况，但有些根据 Scope Selector 的 snippets 可能会失效；
  2. 支持属性选择器 (Attribute Selector) ；
  3. 支持 Sass 专用语句，包括 Variable, Directive, Directive Shorthand (= +), Placeholder Selector, Build-in Function, Interpolation 等等；
  4. 所有 something(…) 将当做 function 结构处理，包括某些 CSS 属性值，Sass 自带函数，Compass/Bourbon 函数，自定义函数等等，但排除了 url(), format(), attr()；
  5. 优化了 @media 高亮效果；
  6. 支持运算符高亮；
  7. 输入 id，class，placeholder 选择器时禁用 snippets；
  8. 独立支持 SCSS 与 Sass 两种语句结构；
  9. 兼容大多数 color scheme。

更详细的解释请查看 https://github.com/P233/Syntax-highlighting-for-Sass，如果遇到问题欢迎提交 issues 或者 pull request。

此外，基于 Syntax Highlighting for SCSS 制作了 Syntax Highlighting for CSS 用于替换 ST 默认的 CSS 高亮包，主要优点是完美高亮所有的属性值与属性名，下载地址：https://github.com/P233/Syntax-highlighting-for-CSS
