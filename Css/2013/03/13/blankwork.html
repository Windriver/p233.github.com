【分享】Blankwork 最棒的网格系统

最近有点忙，没有坚持写博客很不应该，今天与大家分享一款 SCSS Grid System —— Blankwork，这是我所有了解过的网格系统中最牛也是最好用的一款，与其他相比不需要定义任何一个额外的 class，通过调用自定义函数与 maxin 直接给元素编译样式，所以强烈推荐所有 Sass 用户尝试一下。地址：http://www.blankwork.net/

首先，导入 blankwork.sass 文件，这个文件中有几个参数可以修改，默认宽度 960px，12列布局，间距 20px。

$_columns: 12 !default;      // Total number of columns
$_column-width: 60px !default;   // Width of a single column
$_gutter: 20px !default;     // Width of the gutter

导入后就可以在主文件中调用 Blankwork 的 function 与 mixin 了，主要包含 5 个功能，分别是 wrapper, container, column, get_width, 以及 clearfix。接下来我会一一介绍，同时我会使用 @includ 的简写也就是 + (@mixin 的简写是 =)。

1. +wrapper($columns, $center, $subtract)

Wrapper 定义最外层的宽度，有三个参数可用: 列数（默认：12），居中（默认：true），偏移量（默认 0）。以默认值为例：

#demo
	+wrapper

输出

#demo {
  width: 960px;
  margin-left: auto;
  margin-right: auto;
  *zoom: 1; }
  #demo:before, #demo:after {
    content: "";
    display: table; }
  #demo:after {
    clear: both; }

可以看到输出样式是 12 列的总宽度也就是 960px，居中效果，并且清除浮动。这里主要用到了前两个参数，第三个参数“偏移量”是什么意思呢？如果接触过 960 或者其他网格系统，一定知道使用 border 与 padding 时需要额外的处理，很麻烦，“偏移量”参数就是为了简化这个步骤设计的。例如添加 1px 的边框，需要在总宽度上减去 2px ：

#demo
	+wrapper(12,true,2)
	border: 1px solid black

或者:

#demo
	+wrapper(12,true,22)
	padding 10px
	border: 1px solid black

输出宽度是 960px 减去“偏移量”，算上 padding 与 border 总宽度还是 960px。另外，建议在 blankwork.sass 文件中调整一下偏移量与居中的顺序，不然要多打一次 true。

2. +container($columns, $subtract)

Container 使用两个参数：列数（默认：12），偏移量（默认：0），作用是允许嵌套列。最外层 wrapper 中需要继续向下嵌套列时，就要用这个 mixin 了，而且使用 container 就不需要再使用 alpha, omega 了，超级方便！编译效果如下：

#demo
	+container(12)

输出：

#demo {
  display: inline;
  float: left;
  width: 960px; }

3. column($columns, $subtract, $margin-override)

Column 使用三个参数：列数（默认：12），偏移量（默认：0），间距（默认：20px，每侧各 10px）。前两个相信不用介绍了，第三个参数调左右的 margin 值，与 960 的 prefix suffix 单向 padding 不同，参数值除以二平均分配给两侧，相信不会经常用到。

4. get_width($columns, $onlyInnerWidth)

Get_width 是个自定义函数，用于提取宽度给其他属性使用。第一个参数设定列数（默认：12），第二参数决定提取的宽度是否包含间距，默认：true，也就是不包含。用法如下：

#demo
  width: get_width(3)
	marging-left: get_width(3,false)

5. clearfix

清除浮动： 

#demo
  +clearfix

使用 blankwork 的这段时间省心省力，确实非常不错，如果以后支持响应式设计就更完美了！
